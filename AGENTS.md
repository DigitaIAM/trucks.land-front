# trucks.land-front ‚Äî agent guide

## Quick start

```sh
npm install
npm run dev          # Vite dev server on :5173
npm run check-build  # type-check + build
npm run lint         # eslint . --fix
npm run format       # prettier --write src/
```

## Non-obvious scripts

| Command | What it does | Why it matters |
|---|---|---|
| `npm run check-build` | `vue-tsc --build` then `vite build` | `npm run build` skips type-checking (runs `build-only` in parallel via `run-p`). Use `check-build` before CI-like verification. |
| `npm run type-check` | `vue-tsc --build` | Runs across project references (`tsconfig.app.json` + `tsconfig.node.json`), which is stricter than a single `tsc` pass. |
| `npm run deploy` | `gh-pages -d build` | **Stale.** CI uses `./dist`; this script outputs to `dist/` but deploys from `build/`. Don't use it ‚Äî rely on CI. |

## Dev server quirks

- CORS origin whitelist: `http://localhost:5173`, `https://api.zeptomail.com`
- Proxy at `/zeptomail` ‚Üí `https://api.zeptomail.com` (rewrites path prefix)
- `.env` contains Supabase credentials (`VITE_SUPABASE_URL`, `VITE_SUPABASE_KEY`); these are secrets in CI

## Auto-imports (no explicit imports needed)

- **Vue APIs** ‚Äî `ref`, `computed`, `watch`, `reactive`, `defineComponent`, etc.
- **Vue Router** ‚Äî `useRouter`, `useRoute`, `RouterLink`, `RouterView`
- **VueUse** ‚Äî `useColorMode`, `computedAsync`, `debouncedWatch`, `useLocalStorage`, etc.
- **Stores** ‚Äî `useXxxStore()` from `src/stores/` (e.g., `useAuthStore()`, `useOrdersStore()`)
- **Composables** ‚Äî any export from `src/composables/`
- **Models** ‚Äî any export from `src/models/`
- **Vue macros** ‚Äî `$ref`, `$computed`, `$shallowRef` etc.
- All **components** under `src/components/` are globally registered (PascalCase, e.g. `<Button>`, `<Card>`, `<Modal>`)

## Architecture

- **Vue 3 SPA** with file-based routing (`src/pages/` structure maps to URLs) and layouts (`src/layouts/`, applied via `<route lang="yaml">` blocks in page files)
- **Pinia** stores in `src/stores/` ‚Äî auto-imported, naming: `useXxxStore`
- **Supabase** for auth and real-time DB (`src/composables/use-supabase.ts` provides `supabase` client + channel subscriptions for `orders`, `order_stages`, `order_events`)
- **DaisyUI** (Tailwind CSS v4) ‚Äî widgets in `src/components/widgets/` mirror DaisyUI classes (e.g. `<Button primary>`, `<Card>`, `<Modal>`)
- **VueDatePicker** from `@vuepic/vue-datepicker` ‚Äî registered globally as `<VueDatePicker>` in `src/main.ts`
- **SVGs** in `<template>` importable as Vue components via `vite-svg-loader`
- **Icons** via `unplugin-icons` (Iconify) ‚Äî `<Icon name="mdi:home" />`

## Path aliases

- `@/` and `~/` both ‚Üí `src/`

## Lint / format

**Order: lint ‚Üí format** (ESLint with `@vue/eslint-config-prettier/skip-formatting` means ESLint does not format ‚Äî Prettier handles that separately).

- `npm run lint` ‚Äî ESLint flat config (`eslint.config.ts`), targets `**/*.{ts,mts,tsx,vue}`, ignores `dist/`, `dist-ssr/`, `coverage/`
- `npm run format` ‚Äî Prettier with no semicolons, single quotes, 100 print width, only `src/`

## Type-check

`npm run type-check` runs `vue-tsc --build` using project references (`tsconfig.json` ‚Üí `tsconfig.app.json` + `tsconfig.node.json`). This is the single truth for type errors ‚Äî do not rely on `tsc` directly.

## Tests

**None.** No test framework, no test files, no vitest/cypress/playwright config. There is nothing to run.

## CI / deploy

- GitHub Actions (`.github/workflows/deploy.yml`): triggered on push to `main`
- `npm ci` ‚Üí `npm run build` (env vars from secrets) ‚Üí `cp dist/index.html dist/404.html` ‚Üí upload to GitHub Pages
- Supabase URL/key: `VITE_SUPABASE_URL`, `VITE_SUPABASE_KEY` as secrets

## Gotchas

- `src/views/` exists but is **empty** ‚Äî not used; pages live in `src/pages/`
- The README is a generic Vite starter template ‚Äî ignore it for project-specific guidance
- All stores, composables, and models are auto-imported; do not manually import them unless you need a type only
- Pinia stores use `acceptHMRUpdate` for hot reload support
- `typed-router.d.ts` should be included in tsconfig for typed route names
